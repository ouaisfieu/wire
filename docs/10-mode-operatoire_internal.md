# Mode opératoire – Veille et analyse

**Diffusion : Interne**  
**Référence : MO-CVA-10**

---

## 1. Objet

Ce document décrit le déroulement standard d’une **séquence d’analyse**,  
depuis la réception d’une demande jusqu’à la diffusion du produit final.

---

## 2. Étapes principales

1. **Qualification de la demande**  
   - Identifier le demandeur, le contexte, l’échéance.  
   - Clarifier la question centrale et les zones d’incertitude.  
   - Vérifier la légitimité de la demande au regard du périmètre de la CVA.

2. **Cadrage**  
   - Définir le périmètre thématique et temporel.  
   - Identifier les sources disponibles (ouvertes, restreintes, internes).  
   - Valider les contraintes de diffusion (public, interne, restreint).

3. **Collecte**  
   - Utiliser les canaux et outils validés.  
   - Documenter rapidement les sources critiques (date, auteur, fiabilité).  
   - Éviter les redondances et les dérives hors périmètre.

4. **Analyse**  
   - Mettre en évidence les convergences / divergences entre sources.  
   - Distinguer faits, interprétations et hypothèses.  
   - Produire un raisonnement explicite, même simplifié.

5. **Rédaction**  
   - Utiliser un format adapté (note brève, fiche, alerte, synthèse).  
   - Formuler des conclusions **claires et justifiées**.  
   - Si nécessaire, formuler des scénarios plutôt que des prédictions.

6. **Validation et diffusion**  
   - Soumettre la note pour relecture interne (niveau pair / supérieur).  
   - Appliquer le niveau de classification approprié.  
   - Diffuser via les canaux prévus, en limitant les destinataires au besoin.

---

## 3. Usage de FAUXNET dans le cycle

FAUXNET peut être utilisé :

- comme **banc d’essai** pour tester une note sur des données fictives,  
- comme **support de formation** pour nouveaux arrivants,  
- comme **environnement de démonstration** de scénarios de manipulation.

Les documents produits dans FAUXNET doivent être marqués comme tels,  
et ne jamais être confondus avec une production fondée sur des données réelles.
